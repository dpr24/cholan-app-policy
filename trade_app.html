Key Differences Summary

1. Entry Logic — Fundamentally Different

v1 uses a percentage pullback (60–80%) from a recent high as the entry trigger, combined with trendline touch and MACD confirmation via a grid score. The new requirements replace this with Fibonacci retracement zones (50%, 61.8%, 70%) — which are more precise price-based levels rather than a percentage range. Both aim for pullback entries, but Fibonacci is a more specific and professional framing.

2. VWAP Filter — Completely New

v1 has no VWAP-based rule. The new requirements mandate that price must be ≥ 20 pts away from VWAP on the opposite side of the trend. This is a strong institutional-level filter ensuring entries are made during genuine retracements, not momentum chasing. This is the most significant new addition.

3. Holding Period — Intraday vs Positional

This is the biggest structural conflict. v1 is designed as an intraday system (2-hour max hold, MIS product type). The new requirements envision a positional/swing system (3-day max hold, trailing SL). These cannot coexist without choosing one approach. A 3-day hold requires NRML ordering via AngleOne, different margin requirements, and overnight risk management.

4. News — Block vs Align

v1 blocks all trades near news events (±60 min / ±30 min). The new requirements use news sentiment to align direction — only trade if news sentiment matches the technical trend. This is an upgrade: instead of avoiding news entirely, you use it as a confirming signal.

5. Stop Loss — Fixed vs Dynamic Fibonacci

v1 uses a fixed SL set at order placement. New requirements use 90% Fibonacci level as SL with a trailing mechanism (10% of premium per hour). This dynamic approach locks in profits more effectively in trending conditions.

6. Capital Scaling — Single vs Multi-tier

v1 allocates a fixed lot size per trade. New requirements introduce three-tier scaling: Partial capital at 50% Fib, Add position at 61.8%, Full capital at 70% — allowing progressive risk management as price reaches deeper retracement zones.

7. Strategy Dropdown + Indicators Checklist Grid — Human-in-the-Loop vs Fully Automated

✅ It IS in System Workflow v1 — and it's central to it.
In v1, this UI is the core user interaction loop. Specifically:

  • Dashboard Loop (Step 2): "Strategy Selection: The user selects a strategy (e.g., PULLBACK). This 'unlocks' the checklist on the backend." and "Indicator Selection: User manually toggles indicators or timeframes in the UI."
  • Calculation Engine: "ConfluenceEngine receives the grid of selected indicators (MACD, Trendline, Pullback)." — the confluence score is calculated from whatever the user has toggled ON.

In v1, the human is actively in the loop — they choose which strategy and which indicators to validate. The bot then scores based on that custom selection every 10 seconds.

❌ It is NOT mentioned in the New Requirements.
The new requirements are fully automated. There is no user toggle:

  • Trend is auto-detected from slope on 1H / Daily / Weekly
  • MACD is auto-checked across 4 timeframes (min 2 of 4)
  • Fibonacci levels are auto-calculated from swing high/low
  • VWAP filter runs automatically
  • The engine loops every 5 minutes. no manual human input

The new requirements assume the bot decides everything — there is no checklist for the user to interact with.

✅ Integration Possibilities (What Can Be Added to v1)

The following elements from the new requirements can be integrated directly into v1 without restructuring its core:

  1. Replace Pullback (60–80%) with Fibonacci Zones (50%, 61.8%, 70%)
    In EntryValidator, swap the percentage-pullback check with a Fibonacci level proximity check. Same concept, more precise trigger. No architectural change needed.
  2. Add VWAP Filter as a New Mandatory Pillar
    In the Confluence Engine, add VWAP_RETRACEMENT must be VALID as a fourth mandatory pillar alongside TREND_DIRECTION, MARKET_STRUCTURE, KEY_LEVELS. Calculate VWAP from 15M candles.
  3. Upgrade MACD to Multi-Timeframe (1H, 30M, 45M, 15M)
    In the existing MACD confluence logic, extend to check all 4 timeframes and require ≥ 2 of 4 to align. Backward-compatible addition to the EntryValidator.
  4. Add News Sentiment Alignment on top of the Existing Hard Block
    Keep the existing ±60/30 min hard block, but add a VADER sentiment check during non-restricted windows. Only allow entry if sentiment matches trend direction.
  5. Add 36-Candle Range Check to EntryValidator
    Calculate the 36-candle high/low and validate that entry price is within ±22 pts of the range boundary. Add as a new validation step before order placement.
  6. Upgrade SL to Dynamic Fibonacci 90% + Trailing
    Replace the fixed SL with fib_zones["90"] at entry, then implement trailing logic in monitor_positions() — move SL by 10% of premium each hour.
  7. Add Partial / Add / Full Capital Scaling to CapitalManager
    Check which Fibonacci level triggered the signal, and apply the corresponding capital fraction: 12.5% for Partial, 12.5% for Add, 12.5% for Full.
  8. Add S/R-Based Profit Booking (after ≥ 15 pts premium gain)
    In monitor_positions(), if unrealised premium profit ≥ 15 pts, check if spot price is near a pivot/S/R level on any of 15M/30M/1H/Day/Weekly. If yes, trigger early exit.
  9. Replace Daily-Loss Circuit Breaker with SL-Count Cooldown
    Replace the daily-loss counter with a consecutive-SL counter. At 2 consecutive SL hits, activate a 5-day cooldown. This is a compatible enhancement.
  10. Relax Expiry Minimum from 21 to 12 Trading Days
    Update the expiry filter in Contract Selection from MIN_EXPIRY_DAYS = 21 to MIN_EXPIRY_DAYS = 12. Simple config change.

Full System Workflow: Nifty Trading Bot

This document outlines the complete, end-to-end workflow of the trading system, from raw market data ingestion to final trade exit.

1. Data Layer: Ticks to Candles

  1. MarketWebSocket: Connects to Angel One's SmartAPI. Receives raw "ticks" (price updates) every second.
  2. CandleService: Receives ticks and aggregates them into OHLC (Open, High, Low, Close) bars.
  3. Timeframe Management
    • 1-Minute (1M): The base unit.
    • Multi-Timeframe (MTF): 1M candles are used to build 5M, 15M, 30M, 1H, and Daily (D1) bars.
  4. Redis Storage: All candles are cached in Redis for fast access by the indicator engines.

2. Dashboard Loop: User Interaction

  1. Connection: Client connects to the Node.js server (Port 3001).
  2. Strategy Selection: The user selects a strategy (e.g., PULLBACK). This "unlocks" the checklist on the backend.
  3. Indicator Selection: User manually toggles indicators or timeframes in the UI.
  4. Indicator Update
    • Every 10 seconds, the server runs a Broadcast Loop.
    • It re-calculates indicators using the latest market data.
    • It pushes an INDICATOR_UPDATE message to the UI with a Confluence Score.

3. Calculation Engine: Confluence Analytics

  1. ConfluenceEngine: Receives the grid of selected indicators (MACD, Trendline, Pullback).
  2. EntryValidator: The math core. It checks:
    • MACD: Crossings or divergence.
    • Trendlines: If the current price is near or bounced off a line.
    • Pullback: If price has dropped 60-80% from a recent peak.
  3. Weighting: Each "Valid" cell adds to the total score.
  4. Threshold: A setup is "Tradeable" only if the Confluence Score is above the defined threshold (usually 70%).
  5. News Guard: Even if the score is 100%, the NewsService blocks the trade if high-impact news is imminent (within 60 mins).

4. Execution Layer: Risk & Entry

  1. StrategyExecutor: Main Loop runs every 1-5 seconds.
  2. Confluence Check: Automatically evaluates the user's active strategy and selections.
  3. Risk Pre-Check
    • Time Gate: Only trades during market hours (e.g., 9:30 - 15:00).
    • Circuit Breaker: Stops if the daily loss limit is hit.
    • Cooldown: Prevents trading too quickly after a loss.
  4. Position Sizing: The CapitalManager calculates how many lots to buy based on the active Risk Profile.
  5. Broker Order: Placed via Angel One's API. Typically an Option Buy (ATM/ITM strike).

5. Exit Management: Profit Booking

  1. Target (TP): Closes the position at a predefined profit percentage.
  2. Stop Loss (SL): Triggers an immediate market exit if the price drops below the safety limit.
  3. Time-Based Decay
    • Intra-day: Target reduces after 3 hours (e.g., from +40 to +35 points).
    • Inter-day: If held overnight, the target is reduced by 0.7x decay factor per trading day.
  4. Broadcast: Final "Trade Closed" results are shared back to the UI for logging and performance metrics.
