Full System Workflow: Nifty Trading Bot

This document outlines the complete, end-to-end workflow of the trading system, from raw market data ingestion to final trade exit.

1. Data Layer: Ticks to Candles

  1. MarketWebSocket: Connects to Angel One's SmartAPI. Receives raw "ticks" (price updates) every second.
  2. CandleService: Receives ticks and aggregates them into OHLC (Open, High, Low, Close) bars.
  3. Timeframe Management
    • 1-Minute (1M): The base unit.
    • Multi-Timeframe (MTF): 1M candles are used to build 5M, 15M, 30M, 1H, and Daily (D1) bars.
  4. Redis Storage: All candles are cached in Redis for fast access by the indicator engines.

2. Dashboard Loop: User Interaction

  1. Connection: Client connects to the Node.js server (Port 3001).
  2. Strategy Selection: The user selects a strategy (e.g., PULLBACK). This "unlocks" the checklist on the backend.
  3. Indicator Selection: User manually toggles indicators or timeframes in the UI.
  4. Indicator Update
    • Every 10 seconds, the server runs a Broadcast Loop.
    • It re-calculates indicators using the latest market data.
    • It pushes an INDICATOR_UPDATE message to the UI with a Confluence Score.

3. Calculation Engine: Confluence Analytics

  1. ConfluenceEngine: Receives the grid of selected indicators (MACD, Trendline, Pullback).
  2. EntryValidator: The math core. It checks:
    • MACD: Crossings or divergence.
    • Trendlines: If the current price is near or bounced off a line.
    • Pullback: If price has dropped 60-80% from a recent peak.
  3. Weighting: Each "Valid" cell adds to the total score.
  4. Threshold: A setup is "Tradeable" only if the Confluence Score is above the defined threshold (usually 70%).
  5. News Guard: Even if the score is 100%, the NewsService blocks the trade if high-impact news is imminent (within 60 mins).

4. Execution Layer: Risk & Entry

  1. StrategyExecutor: Main Loop runs every 1-5 seconds.
  2. Confluence Check: Automatically evaluates the user's active strategy and selections.
  3. Risk Pre-Check
    • Time Gate: Only trades during market hours (e.g., 9:30 - 15:00).
    • Circuit Breaker: Stops if the daily loss limit is hit.
    • Cooldown: Prevents trading too quickly after a loss.
  4. Position Sizing: The CapitalManager calculates how many lots to buy based on the active Risk Profile.
  5. Broker Order: Placed via Angel One's API. Typically an Option Buy (ATM/ITM strike).

5. Exit Management: Profit Booking

  1. Target (TP): Closes the position at a predefined profit percentage.
  2. Stop Loss (SL): Triggers an immediate market exit if the price drops below the safety limit.
  3. Time-Based Decay
    • Intra-day: Target reduces after 3 hours (e.g., from +40 to +35 points).
    • Inter-day: If held overnight, the target is reduced by 0.7x decay factor per trading day.
  4. Broadcast: Final "Trade Closed" results are shared back to the UI for logging and performance metrics.

PROJECT LOGIC BLUEPRINT (Exhaustive System Rules)

I. ENTRANCE GATEWAY (Before Trade)

1. Hard Chronological Rules

if (Time < 09:30 IST) -> Trade Blocked
if (OpenPositions >= 1) -> Trade Blocked
if (ExecutedTradesToday >= 1) -> Trade Blocked
if (CurrentTrendTradeCount >= 3) -> Trade Blocked

2. Confluence Engine Validation

  • Mandatory Pillars (D1 to 30M)
TREND_DIRECTION must be VALID
MARKET_STRUCTURE must be VALID
KEY_LEVELS must be VALID

TotalGreenCells / TotalEnabledCells >= 0.70

ENTRY_PATTERN must be VALID on 15M or 5M timeframe

II. CONTRACT & ORDER SELECTION (The If/Else Logic)

1. Instrument choosing

if (MarketDirection == UP) -> Select CALL Option (CE)
if (MarketDirection == DOWN) -> Select PUT Option (PE)

2. Strike Selection Logic

Formula (CE): Strike = floor(SpotPrice - 150)
Formula (PE): Strike = ceil(SpotPrice + 150)
Rule: Strike must be at least 150 points In-The-Money (ITM)

3. Expiry & Premium Logic

Expiry Target: 21 trading days (~3 weeks)

Selection: If multiple contracts exist, choose the Lower Premium

Filter:
if (OptionPremium >= HighestPremiumOfLast3Days)
-> REJECT CONTRACT

III. ORDER EXECUTION SYSTEM (Step-by-Step)

1. Pre-Order Validation

if (Quantity < 1 OR Quantity > 10000) -> REJECT
if (Price <= 0) -> REJECT

PriceDiff = abs(Price - LTP) / LTP * 100
if (PriceDiff > PRICE_SLIPPAGE_PERCENT)
-> REJECT

Reward = abs(Target - Price)
Risk = abs(Price - SL)

if (Reward / Risk < MIN_PROFIT_RATIO)
-> REJECT

2. Sizing & Lot Rounding

Qty = floor( TradeCapital / (Premium * 25) )
FinalQty = floor(Qty / 50) * 50

if (FinalQty < 50)
-> REJECT

3. Broker Order Parameters

Product Type: MIS
Order Type: LIMIT
Validity: DAY
Transaction: BUY for entry, SELL for exit

4. Retry & Timeout Logic

Max Retries: 3 attempts
Wait Interval: ORDER_TIMEOUT / 2

if (Attempts == 3 AND Success == FALSE)
-> ABORT & LOG CRITICAL ERROR

IV. POSITION MONITORING & MODIFICATION

1. Real-Time SL/Target Updates

if (NewSL > 0 AND NewSL < EntryPrice)
-> PERMIT MODIFICATION

if (NewTarget > 0 AND NewTarget > EntryPrice)
-> PERMIT MODIFICATION

2. Time-Based Holding

if (CurrentTime - EntryTime >= 120 minutes)
-> IMMEDIATE EXIT (TIME)

3. News-Based Adjustments

if (NewsRestrictionStatus == RESTRICTED)

New Trade -> Block Entry

Existing Trade -> Reduce position to 33.3%
floor(CurrentQty * 0.33)

IV. HOLDING & MONITORING RULES

if (TimeElapsedDesdeEntry >= 120 minutes)
-> Trigger TIME exit

if (MinutesToNewsEvent <= 60)
-> Block Entry

if (MinutesSinceNewsEvent <= 30)
-> Block Entry

if (TrendlineBroken == TRUE)
-> Immediate Market Exit

if (CurrentPrice Outside PullbackZone (60-80%))
-> Immediate Market Exit

if (Trend direction != MACD histogram direction)
-> Immediate Market Exit

V. EXIT CONDITIONS

1 Stop Loss -> LTP <= StopLossLevel
2 Target -> LTP >= TargetLevel
3 Time Out -> Duration == 2 Hours
4 News Override -> Technical violation during news window
5 Manual Pulse -> User triggers exit via WebSocket

VI. AUDIT & POST-TRADE

if (TradeResult == LOSS)
-> Activate Cooldown Manager

Capital Update -> TotalExposure variable in Redis

All trades stored in MySQL with validationScore and exitReason

MATHEMATICAL LOGIC SPECIFICATION

1. Indicator Calculations

A. MACD

M = 2 / (Period + 1)

EMA_today = (Price_today - EMA_yesterday) * M + EMA_yesterday

MACD Line = EMA(Close,12) - EMA(Close,26)

Signal Line = EMA(MACD_Line,9)

Histogram = MACD_Line - Signal_Line

B. Pullback & Retracement

Range = HighestHigh - LowestLow

R = ((HighestHigh - CurrentClose) / Range) * 100

if (PullbackCount == 1) -> R >= 70%
if (PullbackCount == 2) -> R >= 60%
if (PullbackCount == 3) -> R >= 55%

Return% =
((CurrentPrice - PullbackLow) /
abs(TrendlinePrice - PullbackLow)) * 100

Must be >= 40%

C. Candle Pattern Filters

Doji:
abs(Close - Open) / (High - Low) <= 0.10

Hammer:
(min(Open,Close) - Low) > (abs(Close - Open) * 2)

AND

abs(Close - Open) > ((High - Low) * 0.1)

Mixed Color Requirement:

Last 6 candles must contain at least
1 candle of opposite color
