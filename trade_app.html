This document contains the 100% complete internal logic, formulas, and if/else conditions of the trading system.

I. ENTRANCE GATEWAY (Before Trade)

1. Hard Chronological Rules

  • Time Block: if (Time < 09:30 IST) -> Trade Blocked.
  • Max Positions: if (OpenPositions >= 1) -> Trade Blocked.
  • Daily Quota: if (ExecutedTradesToday >= 1) -> Trade Blocked.
  • Trend Quota: if (CurrentTrendTradeCount >= 3) -> Trade Blocked.

2. Confluence Engine Validation

  • Mandatory Pillars (D1 to 30M):
    • TREND_DIRECTION must be VALID.
    • MARKET_STRUCTURE must be VALID.
    • KEY_LEVELS must be VALID.
  • Grid Density:
    • TotalGreenCells / TotalEnabledCells must be >= 0.70 (70%).
  • Immediate Trigger:
    • ENTRY_PATTERN must be VALID on 15M or 5M timeframe.

II. CONTRACT & ORDER SELECTION (The If/Else Logic)

1. Instrument choosing

  • if (MarketDirection == UP) -> Select CALL Option (CE).
  • if (MarketDirection == DOWN) -> Select PUT Option (PE).

2. Strike Selection Logic

  • Formula (CE): Strike = floor(SpotPrice - 150).
  • Formula (PE): Strike = ceil(SpotPrice + 150).
  • Rule: Strike must be at least 150 points In-The-Money (ITM).

3. Expiry & Premium Logic

  • Expiry Target: 21 trading days (~3 weeks).
  • Selection: If multiple contracts exist, choose the Lower Premium.
  • Filter: if (OptionPremium >= HighestPremiumOfLast3Days) -> REJECT CONTRACT.

III. ORDER EXECUTION SYSTEM (Step-by-Step)

1. Pre-Order Validation (Mandatory Checks)

Before sending the order to the broker, the system checks:

  • Quantity Check: if (Quantity < 1 OR Quantity > 10000) -> REJECT.
  • Price Reasonability: if (Price <= 0) -> REJECT.
  • Slippage Guard: PriceDiff = abs(Price - LTP) / LTP * 100.
    • if (PriceDiff > PRICE_SLIPPAGE_PERCENT) -> REJECT.
  • Profit Ratio:
    • Reward = abs(Target - Price)
    • Risk = abs(Price - SL)
    • if (Reward / Risk < MIN_PROFIT_RATIO) -> REJECT.

2. Sizing & Lot Rounding

  • Formula: Qty = floor( TradeCapital / (Premium * 25) ).
  • Nifty Lot Rounding: FinalQty = floor(Qty / 50) * 50.
  • Hard Minimum: if (FinalQty < 50) -> REJECT.

3. Broker Order Parameters

  • Product Type: MIS
  • Order Type: LIMIT
  • Validity: DAY
  • Transaction: BUY for entry, SELL for exit.

4. Retry & Timeout Logic

  • Max Retries: 3 attempts (MAX_ORDER_RETRY).
  • Wait Interval: ORDER_TIMEOUT / 2 milliseconds between failed attempts.
  • Failure Action: if (Attempts == 3 AND Success == FALSE) -> ABORT & LOG CRITICAL ERROR.

IV. POSITION MONITORING & MODIFICATION

1. Real-Time SL/Target Updates

  • Modify SL: if (NewSL > 0 AND NewSL < EntryPrice) -> PERMIT MODIFICATION.
  • Modify TP: if (NewTarget > 0 AND NewTarget > EntryPrice) -> PERMIT MODIFICATION.

2. Time-Based Holding

  • Rule: if (CurrentTime - EntryTime >= 120 minutes) -> IMMEDIATE EXIT (TIME).

3. News-Based Adjustments

  • Status Check: if (NewsRestrictionStatus == RESTRICTED):
    • New Trade: Block Entry.
    • Existing Trade: Reduce position to 33.3% (floor(CurrentQty * 0.33)).
  • EntryPrice: Current Last Traded Price (LTP).
  • Stop Loss (SL): EntryPrice * (1 - RiskPercent / 100).
  • Target (TP): EntryPrice * (1 + (RiskPercent * 2) / 100).

IV. HOLDING & MONITORING RULES

1. Time-Based Holding

  • Auto-Exit: if (TimeElapsedDesdeEntry >= 120 minutes) -> Trigger TIME exit.

2. Real-Time News Shield

  • Entry Protection:
    • if (MinutesToNewsEvent <= 60) -> Block New Trades.
    • if (MinutesSinceNewsEvent <= 30) -> Block New Trades.
  • Live Position Protection (During News Window):
    • if (TrendlineBroken == TRUE) -> Immediate Market Exit.
    • if (CurrentPrice Outside PullbackZone (60-80%)) -> Immediate Market Exit.
    • if (Trend direction != MACD histogram direction) -> Immediate Market Exit.

V. EXIT CONDITIONS

A position is closed immediately if ANY of these trigger:

  1. Stop Loss: LTP <= StopLossLevel.
  2. Target: LTP >= TargetLevel.
  3. Time Out: Duration == 2 Hours.
  4. News Override: Technical violation during economic news window.
  5. Manual Pulse: User triggers exit via WebSocket (Dashboard).

VI. AUDIT & POST-TRADE

  • Loss Penalty: if (TradeResult == LOSS) -> Activate Cooldown Manager (adds to consecutive loss counter).
  • Capital Update: Real-time reduction/addition to TotalExposure variable in Redis.
  • Persistence: All trades stored in MySQL database with validationScore and exitReason.

MATHEMATICAL LOGIC SPECIFICATION (Raw Formulas)

This document details every specific mathematical calculation and threshold currently programmed in the system.

1. Indicator Calculations

A. MACD (Trend & Momentum)

  • EMA Multiplier: M = 2 / (Period + 1)
  • EMA_today = (Price_today - EMA_yesterday) * M + EMA_yesterday
  • MACD Line = EMA(Close, 12) - EMA(Close, 26)
  • Signal Line = EMA(MACD_Line, 9)
  • Histogram = MACD_Line - Signal_Line

B. Pullback & Retracement (%)

  • Range = HighestHigh - LowestLow
  • R = ((HighestHigh - CurrentClose) / Range) * 100
  • Zig-Zag Validation:
    • if (PullbackCount == 1) -> R >= 70%
    • if (PullbackCount == 2) -> R >= 60%
    • if (PullbackCount == 3) -> R >= 55%
  • Return% = ((CurrentPrice - PullbackLow) / abs(TrendlinePrice - PullbackLow)) * 100 (Must be >= 40%)

C. Candle Pattern Filters

  • abs(Close - Open) / (High - Low) <= 0.10
  • (min(Open, Close) - Low) > (abs(Close - Open) * 2) AND abs(Close - Open) > ((High - Low) * 0.1)
  • Last 6 candles of confirmation TF must contain at least 1 candle of the opposite color.

2. Confluence Engine Scoring logic

A. Timeframe Weighting (Importance)

B. Scoring Formula

Total_Score = Σ (Valid_Cell_Weight)

Max_Score = Σ (Enabled_Cell_Weight)

Final_Confluence_Score = round((Total_Score / Max_Score) * 100)

3. Order & Execution Math

A. Pre-Order Validations

  • PriceDiff = abs(Price - LTP) / LTP * 100
  • if (PriceDiff > 3%) -> REJECT
  • if ((Target - Price) / (Price - SL) < 2.0) -> REJECT

B. Position Sizing

  • TradeCap = TotalCapital * RiskOption%
  • Qty = floor(TradeCap / (OptionPremium * 25))
  • FinalLotes = floor(Qty / 50)
  • FinalOrderQty = FinalLotes * 50

C. Execution Reliability

  • if (OrderFailed == TRUE) -> Wait timeout / 2 and Retry (Max 3 times).
  • Product forced to PRODUCT.MIS
  • Order type forced to ORDER_TYPE.LIMIT

4. News Safety Window

  • Hard Block Window: [NewsEventTime - 60min] to [NewsEventTime + 30min]
  • if (RestrictionActive == TRUE) -> Block Entry OR Reduce Position to 33.3% (floor(CurrentQty * 0.33))
